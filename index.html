Privacy Policy

SensorApp — Body Intelligence Platform
Effective Date: February 20, 2026

1. Introduction

SensorApp ("we", "our", or "the app") is a health and wellness application that uses your iPhone's built-in sensors to provide real-time body intelligence insights. We are committed to protecting your privacy. This Privacy Policy explains what data the app accesses, how it is used, and how it is protected.

Key Principle: All sensor data, including face data, is processed entirely on your device. We do not collect, transmit, store on servers, or share any personal data with anyone. There is no backend, no cloud storage, no analytics, and no user accounts.

2. Data Collection and Processing

SensorApp accesses the following device sensors for real-time health monitoring. All data processing occurs locally on your device and is held only in temporary memory during active use.

2.1 Camera (Rear)

Used for heart rate measurement via photoplethysmography (PPG). The camera captures light variations from your fingertip to estimate heart rate, HRV, and respiratory rate. No images or video are recorded or stored.

2.2 Accelerometer

Used for seismocardiography (SCG) to detect cardiac vibrations when the phone is placed on the chest. Motion data is processed in real time and not stored.

2.3 Camera (Front — Vision Framework)

Used for body pose detection via Apple's Vision framework to track 19 body joints and assess posture. No images or video are recorded or stored.

2.4 Microphone

Used for voice analysis (speech recognition and sentiment analysis) and environmental sound classification. Audio is processed on-device in real time and is not recorded or stored.

2.5 Barometer

Used to measure atmospheric pressure and relative altitude changes. Data is processed in real time and not stored.

2.6 Haptic Engine

Used to deliver biofeedback patterns through Core Haptics. No data is collected from this feature.

3. TrueDepth Camera and Face Data

This section specifically addresses how SensorApp uses the TrueDepth camera and face data, in accordance with Apple's guidelines.

3.1 What Face Data Is Collected

The app uses Apple's ARKit framework with ARFaceTrackingConfiguration to access the TrueDepth front-facing camera. The app reads the following face data in real time:

  • 52 facial blend shape coefficients — numerical values (0.0 to 1.0) representing facial muscle positions (e.g., eyeBlink, mouthSmile, browDown). These are abstract numerical coefficients, not images or facial geometry.
  • Derived metrics — the app computes emotion labels (e.g., happy, sad, neutral), stress levels, attention scores, and eye openness from the blend shape coefficients using on-device algorithms.

3.2 How Face Data Is Used

Face data is used exclusively for the following purposes:

  • Displaying real-time emotion detection results to the user on screen.
  • Calculating a stress level estimate shown on the dashboard.
  • Providing wellness insights and correlations within the app.

Face data is not used for facial recognition, user identification, authentication, advertising, or any purpose other than the health and wellness features described above.

3.3 Face Data Storage and Retention

Face data is never stored persistently. All face data (blend shape coefficients and derived metrics) exists only in temporary application memory (RAM) while the Face Analysis feature is actively running. The moment the user stops the Face Analysis session or navigates away from the feature, all face data is immediately discarded from memory. No face data is written to disk, saved to files, stored in databases, or persisted in any form.

3.4 Face Data Sharing

Face data is never shared with any third party. Face data is never transmitted off the device. There is no network communication, no server, no cloud storage, no analytics service, and no third-party SDK that receives face data. All processing is performed entirely on-device using Apple's ARKit framework.

3.5 Face Data Retention Period

Face data is retained for zero seconds beyond active use. Data exists only in volatile memory during an active Face Analysis session and is discarded immediately when the session ends. There is no persistent storage of face data whatsoever.

4. Apple HealthKit Integration

The app offers optional integration with Apple HealthKit to read health summaries (steps, calories, distance). HealthKit access is requested at runtime and requires your explicit consent. The app may write sensor measurements to HealthKit if you enable the "Auto-Save to HealthKit" setting. HealthKit data is governed by Apple's own privacy protections and is never accessed by us.

5. Data Storage

SensorApp does not use any remote servers, cloud services, databases, or external storage. The only persistent data stored on your device consists of:

  • User preferences — toggle states for settings (haptic feedback, HealthKit sync, refresh rate), stored locally via UserDefaults.
  • Onboarding completion flag — a single boolean value indicating whether you have completed the onboarding flow.

No sensor readings, health data, face data, audio, images, or personal information are stored persistently.

6. Third-Party Services

SensorApp does not integrate any third-party services. There are no analytics tools, advertising SDKs, crash reporters, social media integrations, or external APIs. The app operates entirely offline and on-device.

7. Data Security

Because all data processing occurs on-device and no data is transmitted or stored remotely, there is no risk of server-side data breaches. Sensor data exists only in volatile memory during active sessions and is protected by iOS sandboxing and device-level encryption.

8. Children's Privacy

SensorApp does not knowingly collect personal information from children. The app does not require account creation, does not collect personal data, and does not transmit any data off the device.
